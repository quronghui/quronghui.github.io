计算机基础知识

操作系统基础

OS整体概述

  1. OS作用：提供资源和提供人机交互接口
  2. OS管理资源
    • 硬件资源：CPU、内存、I/O设备；
    • 软件资源：文件管理；

进程介绍

  1. 进程的三态模式：
    • 运行<——>就绪<——阻塞；
    • 运行——>就绪：分时运行不同的进程；
    • 运行——>阻塞：外部程序打断；
  2. 进程的组成：程序、数据集合、PCB进程控制器
    • PCB：进程存在的唯一标识；
    • 考：判断PCB属于哪种数据结构？
      • 线性表：顺序组成；
      • 链接方式：通过指针的方式进行交叉排列；
      • 索引方式：通过建立索引表；
  3. 进程间的先后关系表示——前驱图
    • 前驱图：有向无循环图（由节点和有向边组成）；
    • 前驱关系：（1）并行执行；（2）前面完成后该节点才能执行；
  4. 进程间的互斥关系——PV操作（加锁操作）
    • 原因：当两个程序交叉运行时，出现代码交叉执行的问题。
    • 临界资源（交叉共用的区域）：用信号量表示其起源数量；
    • PV操作：对信号量值进行操作？
      • P操作：S = S-1，当S>=0时，执行该进程；否则阻塞该进程（无资源可用）；
      • V操作：S =S+1，当S>0时，执行该进程；否则将该进程放到就绪队列中等待。
    • 信号量的取值范围 = [ S，(N - S) ]
      • N：进程的个数，S：临界区自愿的数量
      • 若S为负数：则表示有S个进程正在排队，等待唤醒；
  5. 进程间的同步关系——PV操作
    • 生产者——>缓冲区——>消费者
    • 3个信号量的介绍？
      • 同步信号量—缓冲区：empty（P操作）、full（V操作）；（实现同步问题）
      • 互斥信号量—临界资源：mutex；（保证缓冲区一次只能有一个生产者/消费者在缓冲区）
    • 生产者的操作？
      • 生产一个数据；在缓冲区存入一个数据（关键操作）；
    • 消费者操作？
      • 从缓冲区取出一个数据；消费一个数据（关键操作）；
    • PV操作的顺序？
      • 在关键操作前，进行一次P操作
      • 在关键操作后，进行一次V操作；
  6. 考：前驱图 + 信号量的组合？
    • 在前驱图上标注信号量S：顺序是从小到大、前驱图的距离优先；
    • PV操作设置情况？
      • 起始点：只有V操作，唤醒下一个进程；
      • 终节点：只有P操作，只有前面的进程都完成，这个进程才能开始；
      • 中间节点：（对于每一个中间节点都有Input和Output）
        • Input：P操作（阻塞等待）
        • Output：V操作（唤醒进程）

存储管理（内存和外存）

  1. 地址重定位方式？

    • 静态重定位方式：
      • 绝对地址 = 相关地址 + 程序存放的内存的起始地址；
      • 静态映射，连续控件；
    • 动态重定位方式：
      • 绝对地址 = 重定位寄存器的地址（BR）+ 逻辑地址寄存器的地址（VR）
      • 寄存器：当CPU真实寻找时，才进行地址变化；

  2. 存储管理方式——内存

    1. 分区管理：连续性
      • 将用户空间进行划分、存在空白分区；

  3. 分页存储管理：逻辑地址——>物理地址（映射）

    • 将区域分为大小相同的页；
    • 逻辑地址32bit：页号（20bit）+ 页内地址-偏移地址（12bit）
    • 物理地址32bit：块号（20bit）+ 块内地址-偏移地址（12bit）
    1. 页表：用于记录 （页号——块号）对应表
      • 页号 = 逻辑地址(十进制) / 页大小
      • 页内地址 = 逻辑地址(十进制) % 页大小
      • 物理地址 = 块号 * 页大小 + 块内地址（= 页内地址）
    2. 考：逻辑地址与物理地址的映射？
      • 物理地址的计算；
      • 两个进程共享物理页n，择各自进程的物理块号也是n;

  4. 分段管理

    • 根据程序的大小，分为不同的段；
    • 有地址越界问题？

  5. 段页管理

    • 逻辑地址 = 段号 + 页号 + 页内偏移地址；

  6. 虚拟存储

    • 内存中暂时不同的存储在虚拟内存里；用的时候进行调用；

设备管理

  1. CPU和设备的串行/并行工作方式？
    • 程序控制：串行
  2. DMA的工作方式？
    • 在主存与外设之间建立直接的数据通路；

文件存储管理

  1. 文件的查找问题？——通过索引的方式；
    • 直接索引：直接指向物理块；
    • 间接索引：扩大索引能够检索的内容；
      • 一级索引：由索引节点，先指向一个索引表，在指向具体的物理块；
      • 二级索引：由索引节点，先指向一个索引表，在指向一个索引表，最后指向具体的物理块；
  2. 考题？索引的方式
    • 单个文件最大的长度；
    • 不同的块号对应的索引方式；

文件存储设备管理（存储在外存的管理）

  1. 位示图法：二进制表示，0/1分别表示存储 / 空闲;
  2. 考题？某一块存储的位置，以及该位为0/1？
    • 二进制数都是从0开始存储；
