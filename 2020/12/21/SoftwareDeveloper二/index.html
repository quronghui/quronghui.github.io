计算机基础知识

计算机组成架构

  1. 冯诺依曼：
    • 五大部分：输入，输出，控制器、运算器（核心）、存储器；
    • 指令和地址：指令有操作码和地址码组成；
    • 地址总线：进行寻址操作；
  2. 现代计算机：
    • 三大部分：CPU、存储（核心）、I/O
    • CPU：ALU运算单元、控制单元；

考：一条指令的执行过程？

  1. 一条指令要执行的过程？
    • 取指——>解析——>执行；
  2. 控制单元？
    • PC：存储下一条要执行指令的地址；
    • IR：存储即将要执行指令的地址；
    • ID：指令的翻译；
  3. 运算单元？
    • ALU：算数运算和逻辑运算单元；
    • AC：累加寄存器，用来暂存数据；
  4. 存储模块？
    • PC——>MAR：MAR地址寄存器，用于存储地址；
    • MAR——>存储器——>MDR：MDR数据寄存器；
    • MDR——>IR：在IR中存储要执行指令的地址。
  5. CPU两大部件的组成？

主存：地址总线和数据总线

  1. 存储器的结构？
    • 地址——单元（bit）——存储值；
  2. 数据总线：存储数据信息（bit）；
    • 有n个数据，需要n条数据总线；
  3. 地址总线：存储地址信息；
    • 按照一个字的长度进行存储；（8bit，32bit，64bit）都是一个字的长度；
    • 有n个数据，需要（n / 字长）条地址总线；
  4. 考：存储的计算？
    • 通过总线进行数据的读取；
    • 进制的转换、进制数的减法（加1）；
    • byte、Kbyte、Mbyte（1024进制）

存储器：容量和速度

  1. 相联存储：Cache；
  2. 存储器——>Cache——>主存——>磁盘（金字塔形式）；

校验码：计算校验码

  1. 码距概念：任意两位编码的最小距离；

    • 考一个概念：
      • 码距为1时，不能做判断；
      • 码距为2时，可以判断是否有误码；
      • 码距为3时，在判断误码的同时，可以进行纠错；（计算机一般只允许一位出错）
    • 具体的公式介绍：需要在看视频

  2. 奇偶校验码

    • 在编码前加1，使其编码中1的个数为偶数（偶校验）/ 奇数(奇校验)
    • 缺陷：只能判断奇数个位出错，不能判断偶数个位出错；

  3. 海明码：分组交叉校验

    1. 特点

      • 基于奇偶校验的校验码；（为了能够检验偶数位出错）
      • 码距至少为2，才能检错和纠错；
      • 检验码所在位置：2^n的位置；
      • 2^k >= n+k+1 (n位数，k位校验码)

    2. 如何划分组：

      • 数据交叉包含；数据位放在非2^n位的位置；
      • 分组方式：第n组，只要第n位为1便可（保证了交叉）

指令周期

  1. 指令周期（一次执行的过程）：取指、分析、执行；

  2. 指令的执行方式？

    • 顺序执行；
    • 重叠执行；
    • 流水线方式：开始和结束有空白；

高速缓冲存储器—Cache

  1. Cache：解决CPU和主存的速度差异；
  2. 考：Cache内容的替换？
    • 时间局部性：例子——for循环内容不替换；
    • 空间局部性：例子——相连部分不替换。
  3. 主存内容——>Cache中的方法？
    1. 直接映像：主存第0页——>cache中第0页
    2. 全相联映像：任意页映射；
    3. 组相联映像：分组映射
  4. Cache性能？
    • 命中和非命中？考平均命中时间
    • 写回策略？
      • 写回法：写入Cache缓存，不立刻写入主存；
      • 写直达法：Cache和主存同时修改；
      • 标记法：双向判断修改；
  5. Cache替换算法
    • 随机替换；FIFO；
    • LRU：最近不使用替换算法；增加计数器
    • LRU：最不经常使用页置换；

磁盘存储器—外存

  1. 磁盘存储器存储方式？
    • 磁盘：两面存取；
    • 磁道——>扇区；
    • 柱面：相同磁道组成的一组柱面；
  2. 考题？——计算存储容量和存取时间？
    1. 存储容量 = N(多少面) T(磁道) S(扇区) * B(字节)
      • 物理块 = 扇区；
    2. 存取时间 = 寻道时间 + 等待时间 + 读/写时间
      • 磁盘顺序处理：磁盘一直再转；
    3. 计算的时候：和磁头号无关，先柱面后扇区号；

计算机系统结构的分类

  1. 体系结构的分类——Flynn（数据流和指令流进行划分）
    • SISD：单指令流单数据流；（单核、顺序执行）
    • SIMD：单指令流多数据流；（多核、并行执行）
    • MISD：多指令流单数据流；（用的最少的：单核、顺序执行）
    • MIMD：多指令流多数据流；（多核、并行执行）
  2. 指令系统：考区别
    • CISC：复杂指令系统；
      • 特点：指令增多、使用频率少、寻址方式多；
    • RISC：精简指令系统；
      • 指令的寻址方式少；
      • 硬布线逻辑控制为主，组合逻辑电路构建控制器；
      • 使用流水线技术；

总线

  1. 定义：是一组能为多个部件分时共享的公共信息通路；
  2. 总线分类：
    • 内部、外部：相对于CPU、芯片等；
    • 按照功能划分：地址、数据、控制总线；
    • 按照数据多少：并行—短距离、串行—长距离；
  3. 考：单工、半双工、全双工 的区别？
    • 单工：方向是单向的；
    • 半双工：两个方向上数据可以轮流传递信息；
    • 全双工：两个方向上同时传递信息；

磁盘阵列的划分

  1. RAID0：将数据分散到多个磁盘上存取
    • 无冗余、无校验；
    • 最高的存储性能；
  2. RAID1：将数据完全一致分配到两块磁盘上；
    • 磁盘镜像阵列，有冗余、无校验；
    • 50%的磁盘空间利用率；
  3. RAID2：采用海明码校验的磁盘阵列
    • 校验盘：提供单纠错和双检错功能；
  4. RAID3/RAID4：交叉校验‘
  5. RAID5：无独立校验盘的奇偶校验磁盘；
    • 校验码：分散到不同的磁盘中；
    • n块磁盘利用率：用户能利用的存储空间为（n-1）
    • 存储容量：以容量最小的盘为准；
  6. RAID6：除了奇偶校验，增加了另一套校验方式；
    • n块磁盘：用户能利用的存储空间为（n-2）
  7. RAID7：加入存储计算机，非磁盘
  8. RAID1+0：是RAID1和RAID0的结合；
    • 存储性能和可靠性最高；
