PyQt 5 信号与槽机制

信号与槽的介绍

  1. QT通信的两种方式？

    • 回调函数Callback；

    • Signal-Slot机制：对象间进行通信的机制；

  2. Signal-Slot之间的特点？

    • 一个信号可以连接多个槽函数，也可以连接另外一个信号；
    • 一个槽函数可以监听多个信号；
    • 信号与槽的连接可以跨线程；

  3. 自定义信号的创建？

    • 使用pyqtSignal()函数：创建一个或多个重载的未绑定的信号作为类的属性；
    • 信号的创建：只能在QObject的子类中定义，并且需要在类创建时定义，不能后期添加；
    • 信号创建：可以携带参数，参数是Python类型；

  4. 信号与槽函数的两种划分方式？

    • 按照使用进行划分：内置信号与槽函数、自定义信号与槽函数
    • 按照性质划分：内置信号与槽函数、内置信号和自定义槽函数、自定义信号和内置槽函数、自定义信号和槽函数；

自定义参数的传递

  1. lambda表达式
    • click信号：他是没有参数的，但是希望槽函数可以接受参数
    • lambda表达式传递按钮数字给槽函数，也可传递其他东西
    • button1.clicked.connect(lambda: self.onButtonClick(1))
  2. 使用functools中的partial函数
    • button1.clicked.connect(partial(self.onButtonClick, 1))
  3. 建议使用lambda表达式；

事件处理机制入门

  1. 事件和信号与槽的区别？
    • Signal-Slot：对事件处理机制的高级封装；
    • Event：用来创建窗口控件，那么Signal-Slot便是用来对这个窗口控件进行使用。
  2. 事件创建窗口控件的描述？
    • Signal-Slot：对于按键的使用，只关心clicked信号，不用关系按钮是如何接受并处理鼠标点击事件的；
    • Envent：如果要改变一个按钮的行为方式，就需要通过Event实现；
  3. 常见的事件类型？
    • PyQt是对Qt的封装，Qt程序是事件驱动的，它的每个动作都由幕后某个事件所触发；
    • 键盘事件、鼠标事件、拖放事件、滚轮事件、绘屏事件、窗口事件……
  4. 使用事件处理的方法
    • 重新实现事件函数：mousePressEvent()、keyPressEvent()、paintEvent();
    • 重新实现QObject.event()，增加新事件；
    • 安装事件过滤器：只是过滤出自己感兴趣的事件
    • 在QApplication中安装事件过滤器：过滤功能更加强大；

窗口数据传递

  1. 单一窗口的数据传递
    • 控件与控件之间：通过Signal-Slot的方式进行数据的传递；
    • 举列：通过滑块Slider控制LCD控件显示数字；

多窗口的数据传递

  1. 方式一：在自定义对话框之间通过属性传参

    • PyQt提供了一写标准的对话框类Dialog，用于输入数据、修改数据、更改应用的设置；

    • 常见的标准对话框：QFileDialog、QInputDialog、QColorDialog、QFontDialog;

  2. 方式二：信号与槽

    • 多窗口的数据传递，通过子窗口发射信号，主窗口通过槽函数捕捉这个信号；
      （1）子窗口发射的信号有两种：内置信号和自定义信号；
（2）自定义信号：参数类型可以进行自定义；
    • 子窗口的作用便是：
      （1）自定义信号：定义信号，发射信号；
（2）内置信号：定义信号；
    • 主窗口的作用：
      （1）自定义信号/内置信号：定义槽函数，完成槽函数和信号的绑定；
（2）调用子窗口的Dialog对话框窗口，通过.show()将其展示出来；
